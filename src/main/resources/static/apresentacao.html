MiniStock

Sistema de Gestão de Estoque Mobile: Uma Solução Full-Stack com Arquitetura Moderna.

Visão Geral do Projeto

O MiniStock é um projeto acadêmico e prático que demonstra a integração de tecnologias de ponta para criar uma solução completa de gerenciamento de estoque. O objetivo principal é fornecer uma ferramenta móvel eficiente e segura para o controle de inventário, desde o registro de produtos até a visualização de métricas de movimentação.

A aplicação é construída em uma arquitetura **Full-Stack desacoplada**, onde o frontend móvel se comunica exclusivamente com uma API RESTful robusta, garantindo escalabilidade e separação de responsabilidades.

Linha do Tempo de Progresso Detalhada

O desenvolvimento do MiniStock seguiu um cronograma estruturado, evoluindo da concepção à implementação de segurança e infraestrutura, conforme detalhado na Proposta Comercial e Técnica.

Fase 1: Concepção, Design e Prototipagem

Início do projeto com o **Levantamento Detalhado** de requisitos (RFs e RNFs). Criação do **Mapa de Navegação** e **Design das Telas no Figma** para garantir a usabilidade e a experiência do usuário (UX/UI). Configuração dos repositórios e ambientes de desenvolvimento.

Fase 2: Modelagem de Dados e Construção do Backend

Foco na camada de dados e API. **Modelagem do Banco de Dados** (MySQL) com as tabelas **`usuario`**, **`produto`**, **`movimentacao_estoque`**, **`notificacao`** e **`configuracao`**. Desenvolvimento dos **Endpoints RESTful** no Spring Boot para as operações CRUD essenciais, garantindo a persistência dos dados.

Fase 3: Desenvolvimento do Frontend e CRUD Básico

Criação das telas de estoque separadas por categoria (**Bebidas, Perecíveis, Não Perecíveis**) e implementação do **CRUD completo** no Frontend. Desenvolvimento da tela de **Movimentação** para registro de entradas e saídas de produtos.

Fase 4: Segurança, Estilização e Refinamento

Etapa crítica de estabilização: **Correção de Múltiplos Erros 403 (Forbidden)**, ajustando a lógica de validação do **JWT** e o **Interceptor** do Axios. **Estilização Unificada** de todas as telas para um visual coeso. **Remoção de campos desnecessários** (URL da imagem, código de barras) para simplificar o uso.

Fase 5: Dashboard e Infraestrutura (CI/CD)

Implementação do **Dashboard** para visualização de métricas. Criação do **Jenkinsfile** para automação do pipeline de CI/CD (compilação da APK e deploy). Desenvolvimento do **Endpoint `/apresentacao`** no Backend para servir a documentação do projeto.

Fase 6: Finalização e Entrega

Testes de integração e usabilidade. Preparação dos **Entregáveis** (código-fonte, manual do usuário). Compilação final da APK e deploy no ambiente de produção (`academico3.rj.senac.br`), concluindo o projeto.

Funcionalidades Detalhadas

Gestão de Produtos (CRUD)

Implementação completa das operações de Criação, Leitura, Atualização e Exclusão (CRUD) para itens de estoque. O sistema suporta a categorização obrigatória de produtos em três grupos distintos: **Bebidas**, **Perecíveis** e **Não Perecíveis**, otimizando a organização e a busca.

Controle de Movimentação

Tela dedicada para o registro preciso de entradas e saídas de produtos. Essa funcionalidade é crucial para manter a integridade dos dados de estoque e permite a rastreabilidade de todas as transações.

Dashboard Analítico

Um painel de controle visual que apresenta métricas chave do estoque, como total de itens, últimas movimentações e distribuição por categoria, oferecendo uma visão gerencial imediata.

Autenticação e Autorização

Sistema de login e cadastro seguro. A comunicação entre o frontend e o backend é protegida por **JSON Web Tokens (JWT)**, garantindo que apenas usuários autenticados e autorizados possam acessar os endpoints da API.

Arquitetura Técnica e Stack

A escolha da stack tecnológica foi orientada pela necessidade de um desenvolvimento rápido, multiplataforma e com performance de nível empresarial.

Frontend (Mobile)

  • **React Native (com Expo Router):** Permite o desenvolvimento de um único código-base para Android e iOS, acelerando o ciclo de desenvolvimento. O Expo Router simplifica a navegação baseada em arquivos.
  • **Axios:** Utilizado para requisições HTTP assíncronas, com um **Interceptor** configurado para anexar automaticamente o token JWT em todas as chamadas à API.
  • **Context API:** Gerenciamento de estado eficiente para o contexto de Autenticação e Estoque, evitando a complexidade de bibliotecas externas para este escopo.
  • **Estilização Unificada:** Aplicação de um tema visual coeso em todas as telas (Login, Cadastro, Tabs, Estoque), garantindo uma experiência de usuário profissional.

Backend (API RESTful)

  • **Spring Boot (Java):** Framework de microsserviços robusto e amplamente utilizado no mercado, oferecendo alta performance e estabilidade.
  • **Spring Security & JWT:** O pilar da segurança. O backend gera e valida tokens JWT, garantindo a integridade e a confidencialidade dos dados. A verificação de `ISSUER` foi ajustada para flexibilidade de ambiente.
  • **Estrutura RESTful:** Endpoints bem definidos para o consumo do frontend, seguindo os princípios REST para manipulação dos recursos (produtos, movimentações).
  • **Endpoint de Apresentação:** O controlador `PresentationController.java` serve esta página estática (`/apresentacao`), demonstrando a capacidade de servir conteúdo HTML diretamente do backend.

DevOps e Qualidade

  • **Jenkinsfile:** Script de pipeline de CI/CD para automação do pipeline de CI/CD para automatizar a compilação do aplicativo móvel (APK) e o processo de deploy, garantindo entregas rápidas e consistentes.
  • **Git Flow Simplificado:** Uso de repositórios separados (`MINISTOCK-FRONTEND` e `MINISTOCK`) para gerenciar o código-fonte de forma organizada.
  • **Correção de Erros 403:** Resolução de problemas de autenticação e autorização, incluindo ajustes no interceptor do Axios e na validação do JWT, garantindo a comunicação fluida entre as camadas.
